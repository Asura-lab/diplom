\chapter{Дүгнэлт}
\label{sec:conclusion}

\subsection{Судалгааны гол үр дүн}

Энэхүү судалгааны ажлаар Flutter дээрх харагдах хэсэг (frontend) болон Node.js/Express дээрх сервер талын хэсэг (backend)-д суурилсан ``AI Notebook'' системийг бодитоор хэрэгжүүлж, интеграцийн туршилтаар баталгаажууллаа. Гол үр дүнгүүд:

\begin{enumerate}
    \item \textbf{Системийн архитектур}: харагдах хэсэг (frontend), сервер тал (backend), өгөгдлийн сан (database), хиймэл оюуны үйлчилгээ (AI service)-ний уялдаатай модульчлагдсан бүтэц амжилттай бүрдсэн.
    \item \textbf{Гол функцууд}: Бүртгэл/нэвтрэлт (signup/login), тэмдэглэлийн CRUD, сэтгэл хөдлөлийн төлөвийн (mood) тэмдэглэгээ, календарийн харагдац, профайл удирдлага, хиймэл оюун ухаанаар стикер үүсгэх (AI sticker generation) функц бүрэн ажилласан.
    \item \textbf{Аюулгүй байдал}: JWT баталгаажуулалт (authentication) болон завсрын хамгаалалтын модуль (middleware)-тай төгсгөлийн цэгүүд (endpoints) хэрэгжиж, токенгүй нэвтрэлтийг хориглосон.
    \item \textbf{Интеграцийн баталгаа}: харагдах хэсгээс (frontend) API дуудлага хийх, MongoDB-д хадгалах, OpenAI интеграц зэрэг бүрэн урсгал (end-to-end flow) амжилттай туршигдсан.
\end{enumerate}

\subsection{Судалгааны шинэлэг тал}

\begin{itemize}
    \item Flutter + Node.js + MongoDB + OpenAI гэсэн хосолсон технологийн стек (technology stack) дээр бодит аппликейшн хөгжүүлж, судалгааны баримт болгон ашигласан.
    \item Энгийн тэмдэглэл хөтлөх системд (note-taking system) хиймэл оюун ухаанаар стикер үүсгэх (AI sticker generation) боломжийг практик түвшинд нэгтгэсэн.
    \item Судалгааны бүлгүүдийг онолоос архитектур, архитектураас туршилтын үр дүн рүү шаталсан уялдаатайгаар боловсруулсан.
\end{itemize}

\subsection{Хязгаарлалт}

Судалгааны ажлын зарим хязгаарлалтуудыг тодорхойлох нь чухал:

\begin{enumerate}
    \item Мэдэгдлийн (notification) модуль сервер талаас (backend) бодит түлхэх мэдэгдлийн үйлчилгээ (push service)-гээр бүрэн тэжээгдээгүй.
    \item Токен шинэчлэх механизм (token refresh) болон үүрэгт суурилсан эрхийн удирдлага (role-based authorization) зэрэг ахисан хамгаалалт хийгдээгүй.
    \item Сүлжээ тасарсан орчинд эхэлж ажиллах синхрончлол (offline-first synchronization), зөрчил шийдвэрлэх механизм (conflict resolution) оруулаагүй.
    \item Ачаалал/гүйцэтгэлийн туршилтыг (load/performance test) үйлдвэрлэлийн хэмжээний орчинд (production-scale environment) нарийвчилж гүйцэтгээгүй.
\end{enumerate}

\subsection{Цаашдын чиглэл}

Судалгааг цаашид дараахь чиглэлээр хөгжүүлэх боломжтой:

\begin{enumerate}
    \item Түлхэх мэдэгдлийн үйлчилгээ (push notification)-г Firebase Cloud Messaging зэрэг платформтой уялдуулах.
    \item Токен шинэчлэх механизм (token refresh), төхөөрөмжийн сешн удирдлага (session management), аюулгүй байдлын бэхжүүлэлт (security hardening) хийх.
    \item Хиймэл оюун ухаанаар стикер үүсгэх (AI sticker generation) шийдлээс гадна хиймэл оюуны тэмдэглэл хураангуй (AI note summarization), сэтгэл хөдлөлийн шинжилгээ (sentiment analysis) нэмэх.
    \item Үүлэн байршуулалт (cloud deployment: Docker + CI/CD) болон мониторингийн орчин бүрдүүлэх.
    \item Хэрэглэгчийн бодит туршилт хийж хэрэглэгчийн туршлагын хэмжүүрээр (UX metrics) үнэлэх.
\end{enumerate}

\subsection{Эцсийн дүгнэлт}

Энэхүү судалгаагаар харагдах хэсэг (frontend), сервер талын хэсэг (backend), өгөгдлийн сан, хиймэл оюуны үйлчилгээ (AI service) зэрэг олон технологийг нэгтгэсэн ажиллах гар утасны системийг амжилттай бүтээж болохыг нотоллоо. AI Notebook нь практикт ашиглах боломжтой анхны хувилбар бөгөөд академик судалгааг бодит архитектур, интеграцийн баталгаатай уялдуулсан инженерийн үнэ цэнтэй үр дүн боллоо.
