\chapter{Судалгааны сэдвийн онол, өнөөгийн түвшин}
\label{ch:literature}

\section{Дижитал тэмдэглэлийн системийн хөгжил}

Тэмдэглэл хөтлөх програмууд анх энгийн текст хадгалалтын үүрэгтэй байсан бол сүүлийн жилүүдэд хэрэглэгчийн оролцоо, өгөгдлийн уялдаа, төхөөрөмж хоорондын синхрончлол, ухаалаг туслах функцийг агуулсан платформ хэлбэрт шилжиж байна. Ийм системүүдийн чанарыг тодорхойлох гол шалгууруудад:

\begin{itemize}
    \item Хэрэглэгчийн өгөгдлийн аюулгүй байдал,
    \item Мэдээллийн бүтцийн уян хатан байдал,
    \item Мобайл интерфейсийн хүртээмж,
    \item Ирээдүйн өргөтгөл хийх архитектурын боломж,
    \item AI суурьтай нэмэлт функцийн интеграц
\end{itemize}

гэх мэт хүчин зүйлс ордог.

\section{Олон платформт нүүр талын хөгжүүлэлтийн суурь}

Flutter нь Google-ээс хөгжүүлдэг хэрэглэгчийн интерфейсийн хэрэгсэл (UI toolkit) бөгөөд нэг кодын сангаас Android, iOS, вэб (web) зэрэг орчинд аппликейшн гаргах боломж олгодог \cite{flutter_docs}. Виджетэд (widget) суурилсан энэ хандлага нь хэрэглэгчийн интерфейсийг (UI) бүрэлдэхүүн хэсгээр удирдах, дахин ашиглах (reuse), хурдан давтамжтай сайжруулах (iterate) үйл явцад тохиромжтой. Dart хэл нь зэрэгцээ урсгалт хүлээлгийн загвар (async/await) болон хариу үйлдэлт интерфейсийн урсгалд (reactive UI flow) нийцтэй тул сүлжээтэй ажиллах харагдах хэсэгт (frontend) өргөн хэрэглэгддэг \cite{dart_docs}.

Flutter ашигласан судалгаануудын түгээмэл давуу тал:

\begin{itemize}
    \item Хөгжүүлэлтийн хурд өндөр,
    \item Хэрэглэгчийн интерфейсийн (UI) нэгэн төрлийн харагдац,
    \item Төрөлх орчны (native) түвшинтэй ойролцоо гүйцэтгэл,
    \item Төхөөрөмж хоорондын код дахин ашиглалт.
\end{itemize}

Сул талын хувьд платформын тусгай боломжууд руу гүн орж ажиллах үед нэмэлт залгаас (plugin) эсвэл төрөлх орчинтой холбогч давхарга (native bridge) шаардагддаг.

\section{Сервер талын архитектур: Node.js, Express, MongoDB}

Node.js нь хаалтгүй гүйцэтгэлийн (non-blocking), үйл явдалд суурилсан (event-driven) загвар ашигладгаараа оролт-гаралтын (I/O) үйлдэл ихтэй веб системд тохиромжтой \cite{nodejs_docs}. Express хүрээ (framework) нь REST төрлийн хэрэглээний програмчлалын интерфейс (REST API) хөгжүүлэлтийг хөнгөн бүтэцтэйгээр дэмжиж, чиглүүлэлт (route), завсрын хяналтын модуль (middleware), хүсэлтийн амьдралын мөчлөг (request lifecycle)-ийг ойлгомжтой зохион байгуулдаг \cite{express_docs}.

Өгөгдлийн сангийн хувьд MongoDB нь баримт бичигт суурилсан (document-oriented) тул хэрэглэгч ба тэмдэглэлийн өгөгдлийг схемийн хувьсал (schema evolution)-тайгаар хадгалах боломжтой \cite{mongodb_docs}. Тэмдэглэлийн өгөгдөлд сэтгэл хөдлөлийн төлөв (mood), наалт зургийн холбоос (stickerUrl), сануулгын хугацаа (reminderAt) зэрэг нэмэлт талбарууд үе шаттай нэмэгдэхэд энэ загвар давуу ажилладаг.

\section{Баталгаажуулалт ба API хамгаалалт}

JWT (JSON Web Token) нь төлөв хадгалалтгүй баталгаажуулалт (stateless authentication) хэрэгжүүлэх түгээмэл стандарт бөгөөд сервер талд сешн (session) хадгалах шаардлага багатай \cite{jwt_rfc}. Токенд (token) хэрэглэгчийн таних мэдээлэл (жишээ нь userId) агуулах замаар хамгаалалттай төгсгөлийн цэгүүдэд (endpoints) нэвтрэх эрхийг шалгадаг.

Гар утасны аппликейшнд токен хадгалалтыг хамгаалалттай санах ойд (secure storage) хийх нь энгийн дотоод санах ойгоос (local storage) илүү найдвартай гэж үздэг. Судалгаанд Flutter Secure Storage ашиглан токен, профайлын үндсэн мэдээллийг хадгалсан.

\section{AI үйлчилгээний интеграц ба хэрэглээний үнэ цэнэ}

Сүүлийн үеийн үүсгэгч хиймэл оюуны (generative AI) үйлчилгээ нь текст, зураг, дуу, код зэрэг олон төрлийн контент үүсгэх чадвартай болсон. OpenAI-ийн зураг үүсгэх API (image generation API) ашигласнаар хэрэглэгчийн заавар текстээс (prompt) шууд зураг үүсгэх боломжтой \cite{openai_images}. Тэмдэглэл хөтлөх системд (note-taking system) энэ боломжийг нэвтрүүлэх нь дараах ашигтай:

\begin{itemize}
    \item Энгийн текст тэмдэглэлийг визуал контенттой холбох,
    \item Хэрэглэгчийн сонирхол, оролцоог нэмэгдүүлэх,
    \item Аппын ялгарах шинжийг нэмэгдүүлэх.
\end{itemize}

Гэхдээ API түлхүүрийн хамгаалалт, хурдны хязгаарлалт (rate limit), өртөг, алдааны боловсруулалт зэрэг инженерийн асуудлуудыг сервер талын түвшинд (backend level) шийдэх шаардлагатай.

\section{Төсөөтэй бүтээгдэхүүнүүдийн дэлгэрэнгүй харьцуулсан шинжилгээ}

Энэхүү дэд бүлэгт тэмдэглэл (note) болон өдрийн тэмдэглэл (journal) ангиллын төлөөлөх бүтээгдэхүүнүүдийг сонгон, тэдгээрийн албан ёсны баримтжуулалт, функцийн тодорхойлолтод тулгуурлан харьцуулсан шинжилгээ хийв \cite{evernote_help}\cite{notion_help}\cite{onenote_docs}\cite{journey_docs}. Зорилго нь зах зээл дээрх шийдлүүдийн технологийн чиг хандлага, функцийн давхцал, мөн судалгааны системийн онцгой шийдлүүдийг академик байдлаар тодорхойлох явдал юм.

\subsection{Харьцуулалтын шалгуур}

Бүтээгдэхүүнүүдийг дараах шалгуураар үнэлэв:

\begin{enumerate}
    \item Функцийн хамрах хүрээ: тэмдэглэл, зохион байгуулалт, өдөр тутмын хэрэглээний урсгалын бүрэн байдал;
    \item Хиймэл оюуны дэмжлэг (AI support): текст, зураг, ухаалаг туслах функцийн бодит интеграц;
    \item Өгөгдлийн удирдлага: синхрончлол, экспорт, хэрэглэгчийн өгөгдөл эзэмшлийн уян хатан байдал;
    \item Архитектурын өргөтгөх чадвар: API интеграц, модулийн түвшний тэлэх боломж;
    \item Хэрэглэгчийн туршлага: мобайл орчин дахь тасралтгүй, ойлгомжтой хэрэглээний урсгал.
\end{enumerate}

\subsection{Төлөөлөх бүтээгдэхүүнүүдийн шинжилгээ}

\textbf{Evernote} систем нь тэмдэглэл, шошго, хайлт, вэб контент хадгалалт зэрэг уламжлалт тэмдэглэл хөтлөх боломжуудаараа (note-taking features) төлөвшсөн экосистемтэй \cite{evernote_help}. Гэвч хэрэглэгчийн өөрчлөн хөгжүүлэх боломж болон хиймэл оюуны функцийн (AI feature) гүн интеграц харьцангуй хязгаарлагдмал.

\textbf{Notion} нь өгөгдлийн сан, баримт бичиг, хамтын ажиллагааг нэг платформд нэгтгэдэг бөгөөд Notion AI зэрэг функцээр мэдлэгийн менежментэд чиглэсэн давуу тал үүсгэдэг \cite{notion_help}. Харин хувийн өдрийн тэмдэглэлийн (journal) богино, өдөр тутмын гар утасны урсгалд хэт ерөнхий бүтэц үүсэх сул тал ажиглагддаг.

\textbf{Microsoft OneNote} нь олон төхөөрөмжийн синхрончлол, Office орчны интеграц, боловсролын хэрэглээнд тохирсон бүтэцтэй \cite{onenote_docs}. Гэвч хиймэл оюунд суурилсан (AI-based) бүтээлч визуал үүсгэлт болон сэтгэл хөдлөлийн төлөвтэй (mood) уялдсан хэрэглээний урсгал нь гол чиглэл биш.

\textbf{Journey} апп нь хувийн өдрийн тэмдэглэл, сэтгэл хөдлөлийн тэмдэглэгээ, нууцлалд төвлөрсөн (privacy-centered) хэрэглээг онцолдог \cite{journey_docs}. Энэ нь journal туршлагад оновчтой боловч сервер талын түвшний (backend level) нээлттэй өргөтгөл болон тусгай хиймэл оюуны модулийн (AI module) тэлэлт хязгаарлагдмал.

\section{Өмнөх судалгааны хязгаарлалт ба энэхүү ажлын шинжлэх ухаан-практикийн хувь нэмэр}

Өмнөх судалгааны материал болон бүтээгдэхүүний шинжилгээний нэгтгэлээс дараах хязгаарлалтууд ажиглагдсан:

\begin{enumerate}
    \item Монгол хэл дээр баримтжуулсан, гар утасны клиент (mobile client), сервер талын үйлчилгээ (backend service), хиймэл оюуны функцыг нэг хүрээнд интеграцчилсан академик ажил хомс;
    \item Аюулгүй нэвтрэлт, тэмдэглэлийн амьдралын мөчлөг (note lifecycle), AI контент үүсгэлтийг өгөгдлийн урсгал ба дарааллын түвшинд зэрэгцүүлэн баталгаажуулсан судалгаа цөөн;
    \item Онолын үндэслэл, архитектурын шийдэл, туршилтын нотолгоог нэг мөр холбосон хэрэгжилтийн жишээ дутмаг.
\end{enumerate}

Эдгээр хязгаарлалтад хариу болгон энэхүү ажил дараах шинжлэх ухаан-практикийн хувь нэмрийг дэвшүүлж байна:

\begin{itemize}
    \item Flutter дээрх харагдах хэсэг (frontend), Node.js/Express дээрх сервер талын хэсэг (backend), MongoDB, OpenAI үйлчилгээний нэгдмэл архитектурын загварыг боловсруулсан;
    \item Нэвтрэлт, тэмдэглэлийн удирдлага, хиймэл оюун ухаанаар стикер үүсгэх урсгалын (AI sticker generation flow) бүрэн хувилбарыг нэг системд хэрэгжүүлж баталгаажуулсан;
    \item Контекст, нэгдүгээр түвшин, хоёрдугаар түвшний өгөгдлийн урсгалын диаграмм болон дарааллын диаграммаар системийн зан төлөвийг шаталсан аргаар тайлбарласан;
    \item Академик бичвэр, хэрэгжилтийн логик, туршилтын үр дүнг уялдуулсан монгол хэл дээрх хэрэглээний жишээг бүрдүүлсэн.
\end{itemize}

\section{Бүлгийн дүгнэлт}

Энэ бүлэгт дижитал тэмдэглэлийн системийн онолын суурь, Flutter дээрх харагдах хэсэг (frontend), Node.js/Express дээрх сервер талын хэсэг (backend), MongoDB өгөгдлийн сан, JWT хамгаалалт, мөн үүсгэгч хиймэл оюуны интеграцын (generative AI integration) өнөөгийн түвшинг тоймлов. Эдгээрийн уялдааг авч үзэхэд AI Notebook системийг модульчлагдсан архитектураар хөгжүүлэх нь технологийн хувьд оновчтой болох нь харагдаж байна. Дараагийн бүлэгт энэхүү онолын үндэслэл дээр тулгуурласан арга зүй, системийн хэрэгжилтийг дэлгэрэнгүй тайлбарлана.
