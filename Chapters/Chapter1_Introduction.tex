\chapter{Удиртгал}

\section{Үндэслэл, ач холбогдол}

Сүүлийн жилүүдэд гар утасны хэрэглээ огцом өссөнтэй холбоотойгоор өдөр тутмын тэмдэглэл, сэтгэл хөдлөл хянах, өөрийгөө ажиглах төрлийн аппликейшний хэрэгцээ нэмэгдэж байна. Гэвч олон хэрэглээ одоог хүртэл энгийн текст хадгалалт дээр төвлөрсөн, аюулгүй байдал сул, хэрэглэгчийн оролцоог тогтвортой хадгалах интерактив боломж хязгаарлагдмал хэвээр байна.

Олон платформт хөгжүүлэлтийн орчин (cross-platform framework) болох Flutter нь нэг кодын сангаас Android болон iOS орчинд аппликейшн бүтээх боломж олгож, хөгжүүлэлтийн хугацаа ба зардлыг бууруулдаг \cite{flutter_docs}. Нөгөө талаас Node.js болон Express дээр суурилсан сервер талын хэсэг (RESTful backend) нь хөнгөн, өргөтгөх боломжтой, харагдах хэсэг (frontend)-тэй хурдан уялдах давуу талтай \cite{nodejs_docs}\cite{express_docs}. Өгөгдөл хадгалалтын түвшинд MongoDB ашигласнаар тэмдэглэл, хэрэглэгчийн профайл зэрэг бүтэц нь өөрчлөгдөх өгөгдлийг уян хатан загварчлах боломжтой \cite{mongodb_docs}.

Хиймэл оюуны үйлчилгээний хувьд OpenAI-ийн хэрэглээний програмчлалын интерфейс (API)-д суурилсан зураг үүсгэх боломж нэмэгдэх нь энгийн тэмдэглэл хөтлөх аппликейшнийг (note-taking app) хэрэглэгчтэй харилцах бүтээлч платформ болгон өргөтгөж байна \cite{openai_images}. Энэ нь хэрэглэгчийн оролцоог нэмэгдүүлэх, аппликейшний ялгарах давуу талыг бүрдүүлэхэд чухал.

Иймд энэхүү судалгаанд Flutter дээрх харагдах хэсэг (frontend) болон Node.js дээрх сервер талын хэсэг (backend)-ийг ашигласан, JWT хамгаалалттай, хиймэл оюун ухаанаар стикер үүсгэх (AI sticker generation) функцтэй ``AI Notebook'' системийг инженерчлэлийн түвшинд боловсруулж, практикт ашиглах боломжийг үнэлсэн.

\section{Зорилго, зорилт}

\subsection{Судалгааны зорилго}

Энэхүү судалгааны ажлын гол зорилго нь \textbf{Flutter дээрх харагдах хэсэг (frontend) болон Node.js/Express дээрх сервер талын хэсэг (backend)-д тулгуурласан, хэрэглэгчийн баталгаажуулалт, тэмдэглэлийн менежмент, хиймэл оюун ухаанаар стикер үүсгэх (AI sticker generation) боломж бүхий ухаалаг тэмдэглэлийн гар утасны системийг зохион бүтээж, хэрэгжүүлэн, туршилтаар баталгаажуулах} явдал юм.

\subsection{Судалгааны зорилтууд}

Дээрх зорилгод хүрэхийн тулд дараах зорилтуудыг дэвшүүлсэн:

\begin{enumerate}
    \item Ухаалаг тэмдэглэлийн системийн функционал ба функционал бус шаардлагыг тодорхойлох.
    \item Flutter дээр хэрэглэгч төвтэй олон дэлгэцтэй харагдах хэсгийн (frontend) архитектур хэрэгжүүлэх.
    \item Node.js/Express, MongoDB дээр хэрэглэгчийн удирдлага болон тэмдэглэлийн үүсгэх-унших-шинэчлэх-устгах үйлдэл (note CRUD)-ийг гүйцэтгэх сервер талын хэсэг (backend) хөгжүүлэх.
    \item JWT-д суурилсан нэвтрэлт, хамгаалалттай хэрэглээний програмчлалын интерфейсийн (API) харилцааг хэрэгжүүлж баталгаажуулах.
    \item OpenAI-ийн API ашиглан хиймэл оюун ухаанаар стикер үүсгэх (AI sticker generation) модулийг системд нэгтгэх.
    \item Бүрэн урсгалын туршилт (end-to-end test) хийж системийн ажиллах чадвар, өргөтгөх боломжийг үнэлэх.
\end{enumerate}

\section{Судалгааны хамрах хүрээ}

Энэхүү судалгаанд дараах хүрээг баримталсан:

\begin{itemize}
    \item Мобайл талд Flutter (Dart) ашигласан.
    \item Сервер талд Node.js, Express, MongoDB ашигласан.
    \item AI функцэд OpenAI-ийн зураг үүсгэх хэрэглээний програмчлалын интерфейс (image generation API) ашигласан.
    \item Системийн үнэлгээг функцийн туршилт, өгөгдлийн урсгалын баталгаажуулалт, дарааллын диаграммын (sequence diagram) шинжилгээгээр хийсэн.
\end{itemize}

Судалгааны гадна үлдээсэн хүрээ:

\begin{itemize}
    \item Түлхэх мэдэгдлийн үйлчилгээ (push notification service)-г үйлдвэрлэлийн орчинд (production environment) бүрэн байршуулсан хэрэгжилт.
    \item Үүлэн орчны автоматаар нөөц нэмэх механизм (cloud auto-scaling), тасралтгүй нэгтгэл-байршуулалтын шугам (CI/CD pipeline)-ын иж бүрэн DevOps хэрэгжилт.
    \item Аппликейшн түгээлтийн дэлгүүрт (app store) нийтлэх үеийн бизнес, хууль, төлбөрийн интеграц.
\end{itemize}

\section{Судалгааны ажлын бүтэц}

Энэхүү төгсөлтийн судалгааны ажил дараах бүтэцтэйгээр зохион байгуулагдсан.

\begin{enumerate}
    \item Нэгдүгээр бүлэгт судалгааны үндэслэл, зорилго, зорилт, хамрах хүрээг тодорхойлсон.
    \item Хоёрдугаар бүлэгт сэдэвтэй холбоотой онолын суурь, хэрэглэгдэж буй технологийн өнөөгийн түвшинг тоймлосон.
    \item Гуравдугаар бүлэгт системийн арга зүй, өгөгдлийн урсгалын түвшинчилсэн задлан шинжилгээ, дарааллын диаграммуудыг (sequence diagrams) тайлбарласан.
    \item Дөрөвдүгээр бүлэгт хэрэгжилтийн үр дүн, интеграцийн туршилт, баталгаажуулалтын дүнг нэгтгэн харуулсан.
    \item Тавдугаар бүлэгт судалгааны ерөнхий дүгнэлт, хязгаарлалт, цаашдын хөгжүүлэлтийн чиглэлийг тодорхойлсон.
\end{enumerate}
